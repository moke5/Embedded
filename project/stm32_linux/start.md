# 项目开发计划

## 开发策略选择

**推荐：先 STM32 → 后 Linux 网关**

理由：
1. STM32 端是数据源，用网络调试助手模拟网关即可独立开发
2. STM32 涉及更多硬件和外设，调试时间不确定，先攻克难点
3. Linux 网关可以用网络调试助手模拟客户端，独立验证
4. STM32 完成后，Linux 网关可以直接接收真实数据

## 注意事项

1. **STM32 开发**
   - 每完成一个外设驱动，立即用独立测试程序验证
   - FreeRTOS 栈大小要根据实际使用调整，避免栈溢出
   - LVGL 内存需求较大，C8T6 的 20KB SRAM 要合理分配

2. **Linux 开发**
   - 网络模式一定要用桥接，否则 STM32 无法访问
   - epoll 使用边缘触发 (ET) 模式更高效
   - 文件目录 `/var/iotdata` 需要提前创建

3. **调试技巧**
   - STM32 端：多用串口打印，少用调试器
   - Linux 端：`gdb` 或 `valgrind` 检查内存问题
   - 网络：用 `wireshark` 或 `tcpdump` 抓包分析
