## stm32



###  超声波测距模块（HC-SR04）



#### 引脚

| PIN  |        |
| ---- | ------ |
| Trig | 控制端 |
| Echo | 接收端 |

#### 使用

控制口发一个 10US 以上的高电平,就可以在接收口等待高电平输出.
一有输出就可以开定时器计时,当此口变为低电平时就可以读定时器的值,此时就为此次测
距的时间,方可算出距离.如此不断的周期测,就可以达到你移动测量的值了。



#### 现象

HC-SR04超声波传感器接入STM32后，测量数据始终显示为 `0.000000`，无法正常读取距离值。

通过串口调试输出分析，发现以下关键现象：

| 现象                     | 说明                | 可能原因             |
| ------------------------ | ------------------- | -------------------- |
| `Timer value: 0 us`      | 定时器计数值始终为0 | 定时器中断未触发     |
| `Invalid timer value: 0` | 距离计算无效        | 计时失败导致         |
| ECHO信号能被检测到       | while循环能正常退出 | GPIO输入配置基本正常 |



#### 原因

经过深入分析，问题主要集中在以下几个方面：

##### (1) 定时器中断配置问题

**原代码问题**：

```c
// timer.c中存在外部变量引用
extern u16 msHcCount;  // 依赖外部定义
```

**问题分析**：

- `msHcCount` 在 `ultrasonic.c` 中定义，但 `timer.c` 和 `ultrasonic.c` 互相引用，导致**循环依赖**
- 编译器无法正确解析符号，导致中断处理函数无法正确更新计数值

##### (2) ECHO引脚配置问题

**原代码问题**：

```c
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IN_FLOATING;  // 浮空输入
```

**问题分析**：

- HC-SR04的ECHO输出是**集电极开路**，需要上拉电阻才能正确检测高电平
- 浮空输入模式下，引脚电平不稳定，可能导致误判

##### (3) 位带操作兼容性问题

**原代码问题**：

```c
#define TRIG_Send PAout(0)  // 位带操作宏
TRIG_Send = 1;             // 在某些编译器配置下无法编译
```

**问题分析**：

- 位带操作宏在部分Keil MDK版本或优化级别下可能产生编译错误
- 错误信息：`expression must be a modifiable lvalue`

##### (4) 缺少超时机制

**原代码问题**：

```c
while(ECHO_Reci == 0);  // 无限等待，可能卡死
while(ECHO_Reci == 1);  // 无限等待，可能卡死
```

**问题分析**：

- 如果传感器故障或无物体反射，程序会在此处无限循环
- 导致系统卡死，无法继续执行

- 现象



连接不了，使用的是打开热点的手机，在手机浏览器里输入ip地址后（80\81，和未输入）



```
Header fields are too long for server to interpret
```



- 原因

```
浏览器请求头过大，超出了 ESP32 HTTP 服务器的默认解析缓冲区。用**无痕/隐私模式**打开即可（无痕模式不带 cookie，请求头大幅减小）。未解决的话可尝试用更轻量的浏览器（如 Via 浏览器），或修改 `app_httpd.cpp` 中 `max_uri_handlers` / `max_resp_headers` 配置。
```
