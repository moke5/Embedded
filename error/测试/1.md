解析：

A（错误）：fflush(stdin)是未定义行为（C 标准未规定fflush对输入流的作用），不同编译器表现不同，不能依赖其清除输入缓冲区；

B（错误）：printf的缓冲规则：默认行缓冲，但 “遇到\n” 仅触发 “刷新缓冲区的条件之一”，并非 “立即输出”（比如重定向到文件时，\n不一定立即刷盘）；

C（错误）：stderr是行缓冲 / 无缓冲依赖实现（POSIX 标准中stderr默认无缓冲，但 C 标准仅规定 “不要求缓冲”，并非绝对直接输出）；

D（正确）：scanf("%d", &x)会自动跳过输入中的空白字符（空格、换行、制表符），直到读取到数字或非空白字符。

 

5. B 

解析：

A（错误）：零长数组本身不占用内存，无法直接存储数据，需配合动态内存分配（如结构体 +malloc）使用；

B（正确）：零长数组必须作为结构体的最后一个成员（否则会导致结构体成员地址计算错误）；

C（错误）：C99 标准中替换为 “柔性数组”（char arr[];），零长数组是 GCC 扩展，并非 C99 正式支持；

D（错误）：零长数组的长度在编译时为 0，运行时需通过动态分配确定实际可用长度。

```c
#include <stdio.h>

int main() {
    int nums[5];
    int sum = 0;
    float average;
    
    // 从用户输入中读取5个整数并存储到数组中
    printf("请输入5个整数：\n");
    for(int i = 0; i < 5; i++) {
        scanf("%d", &nums[i]);
        sum += nums[i]; // 累加数组元素的值
    }
    // 计算平均值
    average = (float)sum / 5;
    printf("这5个整数的平均值为: %.2f\n", average);
    return 0;
}
```
